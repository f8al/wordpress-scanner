WordPress Scanner (Burp WP) passively finds known vulnerabilities in WordPress plugins and themes while you browse a target through Burp Suite — think of it as a WPScan-style check built into your proxy.

As you browse a WordPress site through Burp, the extension fingerprints the plugins and themes it sees, determines their versions, and matches them against the WPScan Vulnerability Database. Anything vulnerable is raised as a Burp issue — in the extension's own tab, and (with Burp Professional) in the Scanner issues list.

Features
  • Passive detection — no active requests to the target are required; it works from the traffic already flowing through the proxy.
  • Reliable version detection using each plugin's readme.txt (Stable tag / Changelog), not just fragile ?ver= query strings.
  • Optional detection of plugins from wp-admin/admin-ajax.php action calls via a reverse-lookup action database.
  • Intruder payload generators for WordPress plugins and themes, to actively enumerate what passive browsing misses.
  • Configurable scope: detect plugins, themes, or both; scan URLs only or full response bodies; and support for a custom wp-content path.
  • Option to report every known vulnerability for a detected plugin, or to list all discovered plugins even when no vulnerability is known.
  • Local vulnerability database with offline mode — copy the database file to an air-gapped Burp instance and import it.
  • Automatic once-per-day database updates, with integrity-checked downloads.

The vulnerability data is provided by the WPScan Vulnerability Database and requires a free WPScan API token, which you enter in the extension's settings. This build has been updated to use the WPScan Vulnerability Database API v3.
